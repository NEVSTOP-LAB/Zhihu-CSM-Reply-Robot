### 6.1 设计权衡

**问题 18**: cron 每 15 分钟触发 vs 计划文档的 6 小时
- **原因**: 实际需求调整为更频繁的监控
- **影响**: GitHub Actions 额度消耗增加（每月约 6000 分钟）
- **建议**: 保持现状，或根据实际评论频率调整为 30 分钟

**问题 7**: Markdown 引用符号剥离
- **原因**: `_parse_turns` 使用正则 `r'^>\s*'` 剥离引用，可能误伤评论内容
- **影响**: 评论中以 `>` 开头的行会被剥离首字符
- **建议**: 暂时保持，实际使用中未发现误伤案例

### 6.2 低风险遗留

**问题 11**: `summarize_article` content 参数传了 title
- **当前状态**: `run_bot.py:305-308` 仍传递 title 作为 content
- **影响**: LLM 生成的文章摘要基于标题而非正文（因配置中无正文字段）
- **建议**: 未来如需拉取正文，需增加 API 调用

**问题 17**: done 目录路径 `archive/done/` 未在计划中定义
- **当前状态**: `archiver.py:91` 使用该路径
- **影响**: 目录结构不完全符合计划文档
- **建议**: 更新文档，明确 `archive/done/` 用途
