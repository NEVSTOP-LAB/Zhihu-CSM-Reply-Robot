# 费用评估调研

> **调研结论摘要**
>
> 以"每月 300 条新评论、使用 DeepSeek-V3、BAAI/bge 本地 Embedding"为基准场景，
> 月度 LLM API 费用约 **$0.3–$0.6**（人民币 2~4 元），极低。
> GitHub Actions 免费额度完全覆盖所需计算时间。
> 主要变量：评论量、追问深度、是否使用 OpenAI Embedding（vs 本地免费方案）。
> 即使使用 GPT-4o-mini 代替 DeepSeek，月费用也仅约 $0.5–$1.2，仍然很低。

---

## 1. 假设参数（基准场景）

| 参数 | 基准值 | 说明 |
|------|--------|------|
| 监控文章数 | 10 篇 | 知乎专栏文章 |
| 每月新评论数 | 300 条 | 平均每篇 30 条 |
| 追问比例 | 20% | 60 条为追问（有历史上下文） |
| 平均输入 token/次 | 900 | System(400) + RAG(400) + 评论(100) |
| 追问额外 token | +500 | 历史 6 轮上下文 |
| 平均输出 token/次 | 200 | 回复约 150 字 |
| Prompt Caching 命中率 | 60% | 同文章批量处理 |
| CSM Wiki 大小 | 50 页 MD | 约 100 KB |
| GitHub Actions 运行 | 4 次/天 × 30 天 = 120 次 | 每次约 2 分钟 |

---

## 2. 主要成本项

### 2.1 LLM 回复生成（DeepSeek-V3）

**普通评论（240 条）：**
- 输入（缓存命中 60%）：
  - 144 条命中缓存：144 × 900 × $0.07/1M = $0.009
  - 96 条未命中：96 × 900 × $0.27/1M = $0.023
- 输出：240 × 200 × $1.10/1M = $0.053
- **小计：约 $0.085**

**追问（60 条，带历史）：**
- 输入（1400 tokens/条，缓存命中率低 ~30%）：
  - 18 条命中缓存：18 × 1400 × $0.07/1M = $0.002
  - 42 条未命中：42 × 1400 × $0.27/1M = $0.016
- 输出：60 × 200 × $1.10/1M = $0.013
- **小计：约 $0.031**

**LLM 月度合计：约 $0.116**（含少量缓冲）→ **估算 ~$0.15**

---

### 2.2 Embedding（向量化）

#### 方案 A：本地 BAAI/bge-small-zh-v1.5（推荐）

- **成本：$0**（免费本地模型，运行于 GitHub Actions runner）
- 每次 Action 加载模型约增加 30 秒，GitHub Actions 免费额度可覆盖

#### 方案 B：OpenAI text-embedding-3-small

仅需在 Wiki 更新时重新 embedding（增量）：
- CSM Wiki 50 页 × 平均 500 tokens/页 = 25,000 tokens
- 全量重建：25,000 × $0.02/1M = **$0.0005**（几乎忽略不计）
- 每月增量更新（假设 10% 文件变更）：**< $0.0001**

**无论哪种方案，Embedding 费用接近 $0。**

---

### 2.3 GitHub Actions 计算时间

| 步骤 | 时间/次 | 月度总时间 |
|------|---------|-----------|
| checkout + 安装依赖 | 60s | 120 × 60s = 120min |
| 知乎数据抓取 | 30s | 60min |
| RAG 检索 + LLM 调用 | 30s（仅有新评论时） | ~60min |
| 归档 + git push | 30s | 60min |
| **合计** | **~2.5min/次** | **~300min/月** |

- GitHub Actions **免费额度**：Public repo 无限制；Private repo 2000 min/月
- **月度使用约 300 分钟，完全在免费额度内（即使 Private repo）**

---

### 2.4 Wiki 同步（每周一次）

假设每周同步一次 CSM Wiki，增量更新向量库：
- LLM 调用（可选，用于生成摘要）：约 $0.01/次 × 4 次 = $0.04/月
- Embedding（本地方案）：$0

## 4. 敏感性分析

| 变量 | 影响 |
|------|------|
| 评论量翻倍（600 条/月） | 费用约 ×2，仍 < $0.40 |
| 使用 deepseek-reasoner 替换 V3 | 费用约 ×2，约 $0.35 |
| 使用 GPT-4o 替换 DeepSeek | 费用约 ×25，约 $4-5/月 |
| Prompt Caching 命中率降至 0 | DeepSeek 方案费用约 ×2 |
| Wiki 每天全量重建（用 API Embedding） | 增加约 $0.003/天，约 $0.09/月 |

---

## 5. 费用控制建议

1. **优先使用 DeepSeek-V3**：中文质量优异，费用最低
2. **BAAI/bge 本地 Embedding**：零 Embedding 费用
3. **批量处理同文章评论**：最大化 Prompt Cache 命中率
4. **设置月度 Token 预算告警**：在 DeepSeek/OpenAI 控制台设置用量上限（如 $5/月），超额停止
5. **评论分级处理**：对简单感谢评论（"谢谢！"）跳过 LLM 调用，直接回复模板
