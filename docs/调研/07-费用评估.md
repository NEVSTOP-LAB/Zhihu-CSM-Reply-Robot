# 费用评估调研

> **调研结论摘要**
>
> 以"每月 1000 次问答、使用 DeepSeek-V3、BAAI/bge 本地 Embedding"为基准场景，
> 月度 LLM API 费用约 **$0.4–$0.8**（人民币 3~6 元），极低。
> 主要变量：问答量、多轮对话深度、是否使用 OpenAI Embedding（vs 本地免费方案）。
> 即使使用 GPT-4o-mini 代替 DeepSeek，月费用也仅约 $1.5–$3，仍然很低。

---

## 1. 假设参数（基准场景）

| 参数 | 基准值 | 说明 |
|------|--------|------|
| 每月问答次数 | 1000 次 | 平均使用量 |
| 多轮对话比例 | 20% | 200 次携带历史上下文 |
| 平均输入 token/次 | 900 | System(400) + RAG(400) + 问题(100) |
| 多轮额外 token | +500 | 历史对话上下文 |
| 平均输出 token/次 | 200 | 回答约 150 字 |
| Prompt Caching 命中率 | 50% | 相同 System Prompt 前缀 |
| CSM Wiki 大小 | 50 页 MD | 约 100 KB |

---

## 2. 主要成本项

### 2.1 LLM 问答生成（DeepSeek-V3）

**单轮问答（800 次）：**
- 输入（缓存命中 50%）：
  - 400 次命中缓存：400 × 900 × $0.07/1M = $0.025
  - 400 次未命中：400 × 900 × $0.27/1M = $0.097
- 输出：800 × 200 × $1.10/1M = $0.176
- **小计：约 $0.30**

**多轮对话（200 次，带历史）：**
- 输入（1400 tokens/次，缓存命中率低 ~30%）：
  - 60 次命中缓存：60 × 1400 × $0.07/1M = $0.006
  - 140 次未命中：140 × 1400 × $0.27/1M = $0.053
- 输出：200 × 200 × $1.10/1M = $0.044
- **小计：约 $0.10**

**LLM 月度合计：约 $0.40**

---

### 2.2 Embedding（向量化）

#### 方案 A：本地 BAAI/bge-small-zh-v1.5（推荐）

- **成本：$0**（免费本地模型）
- 首次加载模型约需几秒，后续无额外开销

#### 方案 B：OpenAI text-embedding-3-small

仅需在 Wiki 更新时重新 embedding（增量）：
- CSM Wiki 50 页 × 平均 500 tokens/页 = 25,000 tokens
- 全量重建：25,000 × $0.02/1M = **$0.0005**（几乎忽略不计）
- 每月增量更新（假设 10% 文件变更）：**< $0.0001**

**无论哪种方案，Embedding 费用接近 $0。**

## 4. 敏感性分析

| 变量 | 影响 |
|------|------|
| 问答量翻倍（2000 次/月） | 费用约 ×2，仍 < $0.80 |
| 使用 deepseek-reasoner 替换 V3 | 费用约 ×2，约 $0.80 |
| 使用 GPT-4o 替换 DeepSeek | 费用约 ×25，约 $10/月 |
| Prompt Caching 命中率降至 0 | DeepSeek 方案费用约 ×1.5 |
| Wiki 全量重建（用 API Embedding） | 增加约 $0.001/次，可忽略 |

---

## 5. 费用控制建议

1. **优先使用 DeepSeek-V3**：中文质量优异，费用最低
2. **BAAI/bge 本地 Embedding**：零 Embedding 费用
3. **设置月度 Token 预算告警**：在 DeepSeek/OpenAI 控制台设置用量上限（如 $5/月），超额停止
4. **相似度阈值过滤**：设置 `similarity_threshold` 避免无关 RAG 片段浪费 token
