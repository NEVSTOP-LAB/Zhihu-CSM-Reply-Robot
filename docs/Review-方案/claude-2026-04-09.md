# PR 方案评审（claude · 2026-04-09）

已完成整体方案评审。以下是综合建议：

## ✅ 方案亮点
**1. 技术选型合理**

* DeepSeek-V3 性价比极高（~$0.20/月），中文质量优秀
* 本地 BGE embedding 零成本，避免 API 依赖
* ChromaDB 本地持久化，适合 Git 管理

**2. 架构设计扎实**

* RAG 而非 Skill：静态知识库场景完全正确
* Thread 对话管理：追问上下文结构清晰
* 真人回复⭐高权重索引：自学习闭环设计优秀

**3. 成本控制到位**

* Prompt Caching 利用充分（缓存命中 -74% 费用）
* 增量更新机制（MD5 哈希）避免重复 embedding
* 批量处理同文章评论最大化缓存命中

## ⚠️ 潜在风险与建议
**1. 知乎写操作限制（高风险）**

* 🔴 问题：Cookie 无法调用写 API（POST /comments），必须 OAuth
* 💡 建议：

  * 阶段一：先实现"人工审核模式"（生成回复→写入 pending/→人工复制粘贴）
  * 阶段二：申请知乎开放平台 OAuth（需审核，周期未知）
  * 备选方案：用 Playwright 模拟浏览器操作（风险高，不推荐）

**2. Cookie 失效频率（中风险）**

* 🔴 问题：知乎 Cookie 有效期 1-6 个月，失效后需人工更新
* 💡 建议：

  * 实现 GitHub Issue 自动告警（检测到 401/403 立即通知）
  * 文档中增加"Cookie 更新流程"（截图教程）
  * 考虑每次运行后记录 Cookie 存活状态到日志

**3. 知乎反爬风控（中风险）**

* 🔴 问题：频繁请求可能触发限流或封号
* 💡 建议：

  * 严格遵守 1-2 秒请求间隔（当前设计已包含）
  * 429 错误指数退避最多重试 3 次（已覆盖）
  * 监控告警：连续 3 次失败自动暂停 workflow

**4. 实施计划过于乐观（低风险）**

* 🔴 问题：阶段时间估算（1-2 周/阶段）未考虑 OAuth 审核、知乎 API 变化等不可控因素
* 💡 建议：调整里程碑为"功能验收"而非时间节点

**5. 边界情况处理缺失**

* 需补充以下场景：

  * 超长评论截断策略（>500 tokens）
  * 敏感词/广告评论过滤（跳过回复）
  * 重复评论检测（相同用户短时间多次评论）
  * 文章被删除/禁止评论时的错误处理
